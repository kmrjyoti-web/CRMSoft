## What was done:

### Builder page (builder/page.tsx) — full redesign:
- LEFT panel: 4-step selector (Partner → Edition → Brand → Vertical) instead of old 6-step
- Master code generated: {PARTNER}_{EDITION}_{BRAND}_{VERTICAL}
- Master code check: GET /pc-config/master-codes with filters, fall back to 'not-found'
- "Master code exists" green banner OR "Create" button (same visual style as old builder)
- ?masterId= URL param: pre-loads master code from list page navigation
- 3-tab right panel:
  - "User Type Configs": card grid (2 cols), each card shows userType, subType, extra fields count, resolvedCode, Edit/Delete buttons
  - "Common Reg Fields": shared fields across all user types — add/remove JSON fields
  - "Common Stages": read-only stage list, edit via API/migration
- Config drawer (right-slide modal):
  - New: user type button grid + sub-type button grid (fetched from /sub-types)
  - Display name input
  - Resolved code auto-preview (readonly)
  - Extra registration fields (add/remove inline)
  - Save: POST or PATCH depending on new/edit

### Combined codes list page (page.tsx) — full redesign:
- Header: "Master Codes — X master codes · Y configurations"
- Filter bar: Partner dropdown, Brand dropdown, Vertical dropdown, Active-only toggle
- Expandable master code rows (chevron expand/collapse)
- Each master row: masterCode, displayName, dimensions (partner·edition·brand·vertical), config count badge, hover "Edit" → Builder
- Expanded view: child config rows (color-coded by user type, resolvedCode, subType, active badge)
- Configs loaded on-demand (lazy) with caching
- Empty state with "Open Builder" CTA
- Legacy link preserved at bottom

### api.ts — 10 new methods:
- masterCodes(filters?) — GET /pc-config/master-codes
- masterCode(id) — GET /pc-config/master-codes/:id
- createMasterCode(dto) — POST
- updateMasterCode(id, dto) — PATCH
- deleteMasterCode(id) — DELETE
- masterCodeConfigs(masterCodeId) — GET children
- createMasterCodeConfig(masterCodeId, dto) — POST child
- updateMasterCodeConfig(masterCodeId, configId, dto) — PATCH child
- deleteMasterCodeConfig(masterCodeId, configId) — DELETE child
- resolvedFields(resolvedCode) — GET merged fields

## Backward compatibility:
- Old /combined-code/:code URL still works (backend unchanged)
- Legacy combined codes link preserved on list page
- Registration fields + onboarding stages on old codes still readable via resolvedCode
