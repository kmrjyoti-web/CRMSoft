## Grep scan results

### 1. Pincode

Existing usage: `pincode` as a **field** (String?) in many models across WorkingDB/DemoDB/IdentityDB/PlatformDB/MarketplaceDB. These are all raw pincode strings stored directly on records.

**Existing model `CompanyPincode`** in `working/v1/config.prisma` and `demo/v1/config.prisma`:
- Has `tenantId` field — this is a **tenant-scoped config model** (which pincodes a business operates in)
- Table name `gv_cfg_company_pincodes` — clearly a WorkingDB business config, not a global reference
- ✅ No conflict with `gl_cfg_pincode` (global reference lookup of all Indian PIN codes)

No existing `Currency` model in any schema. `GlCfgCountry.currencyCode` stores a raw string "INR" — no FK relationship.
✅ Safe to create `GlCfgCurrency`

### 3. Timezone

### 4. HsnCode / HSN

`IndustryPackage` and `IndustryPatch` in `platform/v1/modules.prisma` — these are **platform feature/configuration models** about which modules belong to which industry package. Not a reference list of industry types.
